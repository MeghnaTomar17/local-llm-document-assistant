Table: resumes

Columns:
- id UUID
- resume_hash TEXT
- original_file_name TEXT
- stored_file_name TEXT
- file_path TEXT
- mime_type TEXT
- resume_blob BYTEA
- candidate_name TEXT
- email TEXT
- phone_number TEXT
- skills JSONB
- cities JSONB
- fresher BOOLEAN
- processing_status TEXT
- extraction_status TEXT
- is_verified BOOLEAN
- hr_notes TEXT
- technical_notes TEXT
- final_notes TEXT
- hr_decision TEXT
- decision_at TIMESTAMP
- uploaded_at TIMESTAMP
- updated_at TIMESTAMP

Notes:
- skills is a JSONB array of strings. For keyword matching, use skills::text ILIKE '%keyword%'.
- cities is a JSONB array of strings. For city matching, use cities::text ILIKE '%city%'.
- fresher = true means the candidate is a fresher.
- fresher = false means the candidate has professional experience.
- is_verified = true means a recruiter has verified or edited the metadata.
- hr_decision is one of PENDING, ON_HOLD, ACCEPTED, or REJECTED.
- Use only the resumes table.
